Overview

This minimal application is a Django boilerplate for web applications.


It contains everything you need to get started so that you can start building your functionality straight away.

Getting Started

This project uses Python 3.13 and Django 5.2.
To setup the application, you will need to create a virtual environment and install the requirements:

  • python -m venv venv
  • venv\scripts\activate
  • pip install -r requirements.txt

Copy the example environment variables:

  • cp example.env .env

Create your local database:

  • python app\manage.py migrate

Load the example fixtures:

  • python app\manage.py loaddata app\app_src\fixtures\01_users.json
  • python app\manage.py loaddata app\app_src\fixtures\02_notes.json

Create a local superuser account and answer the prompts:

  • python app\manage.py createsuperuser

Run the application, please ensure you choose a port that is not in use:

  • python app\manage.py runserver 8000

You should now be able to visit the running application at http://localhost:8000

Configuration

The following variables in app\app_site\settings.py can be set to configure application behaviour:

  • APPLICATION_NAME - the name of the application, used throughout the site.
  • APPLICATION_VERSION - the version of the application, indicated in the header/footer.
  • APPLICATION_ENVIRONMENT - the environment in which the application is running.

The following variables in .env can be set to configure application behaviour:

Note: to do this you must first copy the example.env as .env - e.g. cp example.env .env

  • DEBUG - set this to True for local development to see verbose debugging information.
  • SECRET_KEY - define a secret key to prevent it being randomly generated at runtime.
  • APPLICATION_ENVIRONMENT - set the application environment such as Staging or Production (default: local).

Notes Application

This example application allows users to create notes, the home page will display the 5 most recent notes.


A simple API route has also been setup, in this example you can visit the following API endpoint to see the list of notes.

  • http://localhost:8000/api/notes/
